# Konszolidált megközelítés / Consolidated Approach

## Forrás / Source

**Dokumentum:** TOP PLUSZ Közgazdasági és Pénzügyi Útmutató  
**Fejezet:** 2.1.7 Konszolidált megközelítés  
**Oldal:** 14

## Áttekintés / Overview

A konszolidált megközelítés biztosítja, hogy az infrastruktúra tulajdonosa és üzemeltetője közötti pénzmozgások ne torzítsák a projekt pénzügyi megtérülési számításait. Ez különösen fontos, amikor a tulajdonos és az üzemeltető különböző entitások.

---

## Alapelv / Basic Principle

### EU CBA Útmutató (2014) szerint

> "Az EU CBA útmutató (2014) alapján a pénzügyi elemzést általánosságban az infrastruktúra tulajdonosa szempontjából kell elvégezni."

### Konszolidáció szükségessége

> "Ha az infrastruktúra tulajdonosa és üzemeltetője eltér, akkor konszolidált elemzés elvégzése szükséges, hogy az infrastruktúra tulajdonosa és üzemeltetője közti pénzmozgások ne befolyásolják a fejlesztés megtérülési számításait."

---

## Elemzési módszertan / Analysis Methodology

### Lépések / Steps

**1. Külön elemzések:**
> "Ha szükséges, akkor az elemzést először az infrastruktúra tulajdonosa és az üzemeltető szempontjából külön-külön, majd konszolidáltan is el kell elvégezni."

**Folyamat:**
```
Tulajdonos szempontú elemzés
         +
Üzemeltető szempontú elemzés
         ↓
Konszolidált elemzés
(tulajdonos + üzemeltető közti pénzmozgások kiszűrése)
```

---

## Speciális eset: Liberalizált piacok / Special Case: Liberalized Markets

### Probléma / Problem

> "A liberalizált piacoknak köszönhetően gyakran előfordul olyan eset, hogy az infrastruktúrának több üzemeltetője is van, így nehézséget jelenthet a konszolidált elemzés elkészítése."

### Megoldás / Solution

> "Ilyen esetekben az elemzést a Vademecumban foglaltak szerint a projekt kedvezményezettje(i) szempontjából célszerű elvégezni – legyen az akár az infrastruktúra tulajdonosa vagy üzemeltetője, a beruházás típusától függően."

---

## Alkalmazási esetek / Application Cases

### 1. Eset: Tulajdonos = Üzemeltető

**Helyzet:**
- Kedvezményezett maga működteti az infrastruktúrát
- Nincs külső üzemeltető

**Elemzés:**
- Egyszerű pénzügyi elemzés
- Konszolidáció nem szükséges
- Minden költség és bevétel egy entitásnál

**Példa:**
- Önkormányzat saját intézménye működteti a létesítményt

---

### 2. Eset: Tulajdonos ≠ Üzemeltető (egy üzemeltető)

**Helyzet:**
- Kedvezményezett (tulajdonos) szerződést köt egy üzemeltetővel
- Egy üzemeltető van

**Elemzés:**
- Külön elemzés: tulajdonos szempontjából
- Külön elemzés: üzemeltető szempontjából
- Konszolidált elemzés: pénzmozgások kiszűrésével

**Kiszűrendő pénzmozgások:**
- Bérleti díj (tulajdonos → üzemeltető irányban)
- Üzemeltetési díj (tulajdonos → üzemeltető irányban)
- Bevételek átadása (üzemeltető → tulajdonos irányban)
- Egyéb szerződéses kifizetések

**Példa:**
- Önkormányzat (tulajdonos) + vállalkozás (üzemeltető)
- Közszolgáltatási szerződés keretében

---

### 3. Eset: Tulajdonos ≠ Több üzemeltető (liberalizált piac)

**Helyzet:**
- Infrastruktúrának több üzemeltetője van
- Liberalizált piaci környezet

**Elemzés:**
- Vademecum szerint: kedvezményezett(ek) szempontjából
- Tulajdonos VAGY üzemeltető perspektíva
- Beruházás típusától függően

**Példa:**
- Vasúti infrastruktúra: több vasúttársaság használja
- Energetikai infrastruktúra: több szolgáltató

---

## Konszolidációs technika / Consolidation Technique

### Pénzmozgások azonosítása / Identifying Cash Flows

**Tulajdonos → Üzemeltető:**
- Bérleti díj
- Vagyonkezelési díj
- Közszolgáltatási kompenzáció
- Egyéb szerződéses kifizetések

**Üzemeltető → Tulajdonos:**
- Bevételek átadása (ha szerződés szerint)
- Bérleti díj (ha üzemeltető bérli az infrastruktúrát)
- Egyéb kifizetések

### Konszolidációs szabály / Consolidation Rule

**Alapelv:**
```
Konszolidált cash-flow = 
  Tulajdonos cash-flow + 
  Üzemeltető cash-flow - 
  Tulajdonos ↔ Üzemeltető közti pénzmozgások
```

**Cél:**
- Csak a projekt által generált nettó pénzáramok maradjanak
- Belső transzferek ne befolyásolják a megtérülést

---

## Példa: Konszolidált elemzés / Example: Consolidated Analysis

### Kiindulási helyzet:
- **Tulajdonos:** Önkormányzat
- **Üzemeltető:** Vállalkozás (közszolgáltatási szerződéssel)
- **Projekt:** Közösségi közlekedési infrastruktúra

### Tulajdonos szempontú cash-flow:

| Tétel | Összeg (M Ft) |
|-------|---------------|
| Beruházási költség | -1000 |
| Közszolgáltatási kompenzáció (kiadás) | -100/év |
| EU támogatás | +700 |

### Üzemeltető szempontú cash-flow:

| Tétel | Összeg (M Ft) |
|-------|---------------|
| Működési költség | -80/év |
| Jegybevétel | +50/év |
| Közszolgáltatási kompenzáció (bevétel) | +100/év |
| Nettó | +70/év |

### Konszolidált cash-flow:

| Tétel | Összeg (M Ft) |
|-------|---------------|
| Beruházási költség | -1000 |
| EU támogatás | +700 |
| Működési költség | -80/év |
| Jegybevétel | +50/év |
| **Nettó (konszolidált)** | **-30/év** |

**Megjegyzés:** A közszolgáltatási kompenzáció (100 M Ft/év) kiesik, mert belső transzfer.

## Döntési fa / Decision Tree

```
Tulajdonos = Üzemeltető?
    ├─ IGEN → Egyszerű elemzés (konszolidáció nem szükséges)
    └─ NEM → Tulajdonos ≠ Üzemeltető
              │
              ├─ Egy üzemeltető?
              │   ├─ IGEN → Konszolidált elemzés szükséges
              │   │          (tulajdonos + üzemeltető + konszolidált)
              │   └─ NEM → Több üzemeltető (liberalizált piac)
              │              └─ Kedvezményezett szempontú elemzés
              │                 (Vademecum szerint)
```

## Ellenőrzési lista / Checklist

### Konszolidált elemzés előtt:

- [ ] Tulajdonos és üzemeltető azonosítva
- [ ] Szerződéses kapcsolat tisztázva
- [ ] Pénzmozgások azonosítva (tulajdonos ↔ üzemeltető)
- [ ] Tulajdonos szempontú elemzés elkészítve
- [ ] Üzemeltető szempontú elemzés elkészítve

### Konszolidált elemzés során:

- [ ] Belső transzferek azonosítva
- [ ] Belső transzferek kiszűrve
- [ ] Csak nettó projekt cash-flow-k maradtak
- [ ] Konszolidált NPV, IRR számítva
- [ ] Eredmények dokumentálva

### Liberalizált piac esetén:

- [ ] Több üzemeltető azonosítva
- [ ] Kedvezményezett perspektíva választva
- [ ] Vademecum útmutatás követve
- [ ] Beruházás típusa figyelembe véve

---

## Értelmezési kérdések / Interpretation Issues

### Tisztázott:
- ✅ Konszolidáció szükséges, ha tulajdonos ≠ üzemeltető
- ✅ Belső transzfereket ki kell szűrni
- ✅ Liberalizált piac esetén kedvezményezett perspektíva
- ✅ EU CBA Útmutató (2014) alapján tulajdonos szempontú elemzés

### Nyitott kérdések:
- ⚠️ **Több üzemeltető esetén konkrét módszertan:** Részletes útmutatás hiányzik
- ⚠️ **Liberalizált piac definíciója:** Nem definiált pontosan
- ⚠️ **Kedvezményezett választása több üzemeltető esetén:** Döntési kritériumok hiányoznak

## Gyakorlati tanácsok / Practical Advice

### Dokumentálás:
1. Készítsen külön táblázatot tulajdonos és üzemeltető cash-flow-jához
2. Azonosítsa és listázza ki az összes belső transzfert
3. Készítsen konszolidált táblázatot a belső transzferek nélkül
4. Dokumentálja a konszolidációs logikát

### Ellenőrzés:
- Ellenőrizze, hogy minden belső transzfer párosítva van (kiadás ↔ bevétel)
- Győződjön meg róla, hogy a konszolidált összeg logikus
- Hasonlítsa össze a konszolidált NPV-t a külön elemzések eredményeivel

### Kommunikáció:
- Mutassa be mind a három elemzést (tulajdonos, üzemeltető, konszolidált)
- Magyarázza el a konszolidáció logikáját
- Indokolja meg, miért szükséges a konszolidált megközelítés
